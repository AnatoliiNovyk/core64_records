# Changelog 2026-03-17 #01 - DB-first foundation

## Як було
- Публічна сторінка та адмінка працювали напряму через localStorage.
- Пароль адмінки зберігався у фронтенд-даних.
- Контактна форма на сайті лише показувала alert без збереження запиту.
- Backend та БД-шар у проєкті були відсутні.

## Що було зроблено
- Додано спільний data adapter з режимом API-first та fallback на localStorage.
- Перероблено `app.js` для завантаження даних через adapter.
- Перероблено `admin.js` для auth/CRUD через adapter замість прямого localStorage-підходу.
- Оновлено `index.html` (іменовані поля форми, статус відправки, підключення adapter).
- Оновлено `admin.html` (прибрано підказку з дефолтним паролем, підключено adapter).
- Створено backend skeleton на Node.js + Express:
  - health/auth/public/settings/contact/collections маршрути,
  - middleware auth + validation,
  - репозиторій роботи з PostgreSQL.
- Додано міграцію `001_init.sql`, скрипти migrate/seed, `package.json`, `.env.example`.
- Оновлено `README.md` під DB-first архітектуру.

## Що це покращило, виправило, додало
- Покращило архітектуру: фронтенд готовий до роботи через API без повного переписування UI.
- Виправило критичний ризик: пароль більше не зберігається у публічних налаштуваннях сайту.
- Додало реальний шлях до БД: підготовлений backend і структура PostgreSQL.
- Додало керований fallback: сайт може працювати локально навіть якщо API недоступний.
- Додало основу для наступного етапу: міграція/seed/auth/CRUD smoke після налаштування DATABASE_URL.

## Поточний статус
- Реалізація фундаменту завершена.
- Міграція БД поки не виконана через неналаштований `DATABASE_URL`.
