# Floating Release Player Bottom Dock

## Як було

- Release player рендерився як звичайний блок усередині секції релізів.
- При прокрутці сторінки плеєр зникав із зони видимості, і керування відтворенням губилося.
- Кнопки `Back to top` та admin shortcut були жорстко привʼязані до нижнього краю екрана і могли конфліктувати з великим плеєром.

## Що зроблено

- У `index.html` додано стилі плаваючого режиму для `#release-compact-player`:
  - fixed-позиціонування внизу екрана,
  - адаптивна ширина для desktop/mobile,
  - обмеження висоти та внутрішній scroll для контенту,
  - додаткова тінь/підсвітка для відділення від фону.
- Додано CSS-перемінну `--release-player-offset` і привʼязку нижніх відступів утилітних кнопок (`Back to top`, admin shortcut) до цієї перемінної.
- У `app.js` додано керування плаваючим режимом:
  - увімкнення fixed-дока при відкритті player,
  - вимкнення при закритті,
  - динамічний розрахунок висоти плеєра й оновлення `--release-player-offset` через `requestAnimationFrame`,
  - синхронізація офсету при `resize`.
- У розмітці player-контейнера додано клас `release-player-content` для контрольованого overflow в плаваючому режимі.

## Що покращило/виправило/додало

- Плеєр більше не зникає при скролі: керування завжди доступне внизу екрана.
- Контент сторінки не перекривається «в глуху», бо нижній відступ сторінки підлаштовується під реальну висоту плеєра.
- Допоміжні кнопки внизу екрана автоматично зміщуються вище плеєра й залишаються клікабельними.
- UX відтворення став передбачуваним: відкрив плеєр один раз — отримав стабільний floating dock до явного закриття.
