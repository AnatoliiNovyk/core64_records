# Release Player Custom Controls Visualizer And Anti Download UI

## Як було

- У спливаючому release player використовувався нативний `audio` control bar браузера.
- У нативному барі залишалися браузерні дії, які могли відкривати сценарії збереження медіа.
- У плеєрі не було виразної live-візуалізації аудіосигналу в стилі цифрового еквалайзера.

## Що зроблено

- У `index.html` нативний `audio controls` замінено на кастомний блок керування:
  - кнопка play/pause,
  - seek-слайдер позиції,
  - індикатор часу `current / duration`,
  - окремий слайдер гучності.
- Додано canvas-візуалізацію `Digital Spectrum` у межах попап-плеєра.
- У `app.js` реалізовано Web Audio візуалізатор через `AudioContext` + `AnalyserNode` з безперервним рендером бігаючих смуг.
- Додано синхронізацію кастомних контролів із станом відтворення (`play`, `pause`, `timeupdate`, `durationchange`, `ended`, `volumechange`).
- Посилено frontend-hardening проти сценаріїв прямого завантаження з UI:
  - приховано нативні браузерні контролі `audio`,
  - виставляються обмежувальні media-атрибути через JS (`controlsList`, `disablePictureInPicture`, `disableremoteplayback`),
  - заблоковано context menu у зоні керування плеєром/візуалізації.
- Додано i18n-ключі для кастомних контролів у `uk` та `en`.
- Оновлено cache-busting версію підключення `app.js` в `index.html`, щоб новий плеєр гарантовано підхопився після деплою.

## Що покращило/виправило/додало

- Візуальна частина плеєра стала виразнішою: у спливаючому вікні з'явився динамічний «цифровий еквалайзер» з бігаючими доріжками.
- Усунено залежність від нативного `audio`-інтерфейсу, який псував стиль і давав небажані UX-можливості.
- Керування відтворенням тепер повністю інтегроване в дизайн CORE64 і працює послідовно в межах попап-плеєра.
- На рівні фронтенду прибрано прямі UI-сценарії для скачування треків через стандартну панель медіаконтролів.
