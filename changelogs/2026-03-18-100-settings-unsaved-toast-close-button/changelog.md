# Changelog - 2026-03-18 - Settings unsaved toast close button

`settings-unsaved` toast можна було закривати кліком по всій області або клавішами.
Явної кнопки закриття в UI не було, тому дія була менш очевидною для частини користувачів.

Файл: `admin.html`

- Оновлено внутрішню розмітку `#settings-unsaved-toast`:
- додано явну кнопку закриття з іконкою `x`;
- кнопка має `title` і `aria-label` (`Закрити повідомлення`);
- клік по кнопці виконує `dismissSettingsUnsavedToast(event)`;
- додано `event.stopPropagation()`, щоб уникнути дублю обробки кліку контейнером.
- Для тексту повідомлення додано `truncate`, щоб зберегти компактність макета.

Дія закриття toast стала очевиднішою та більш discoverable.
Покращилась доступність для користувачів, які очікують явний контрол закриття.
Поточна логіка життєвого циклу toast не змінена (черга, таймери, a11y-підказки, клавіші).

Статичні перевірки:

- `admin.html`: без помилок
- `admin.js`: без помилок

Smoke API:

- `settingsOk=True; okPage=1; okTotal=5; okItems=1`
