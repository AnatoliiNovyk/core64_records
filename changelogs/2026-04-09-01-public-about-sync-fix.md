# 2026-04-09-01 Public About Sync Fix

## Як було

- В адмінці поля `Опис лейблу` та `Місія` зберігались, але на публічній сторінці About залишались старі значення (`A`/`B`) або виглядало як «порожньо».
- Причина: `settings_i18n` містив застарілі значення, а публічний селект читав їх пріоритетно через `COALESCE(...)`.
- Додатково у фронтенд-рендері About текст оновлювався тільки за truthy-умови, тому шаблонний контент міг «залипати».

## Що зроблено

- У `backend/src/db/repository.js` оновлено `upsertSettingsHeroSubtitles()`:
  - для `settings_i18n` тепер синхронізуються `title/about/mission` та captcha-повідомлення з поточного payload при кожному save;
  - на `ON CONFLICT` оновлюються не лише `hero_subtitle`, а й ці поля.
- У `app.js` (функція `loadAbout`) прибрано умову, яка блокувала оновлення DOM при falsy значеннях:
  - `about` і `mission` тепер завжди перезаписуються актуальним значенням.
- Проведено перевірки:
  - diagnostics для змінених файлів — без помилок;
  - цільовий e2e roundtrip-тест: save settings -> GET `/api/public?lang=en|uk` повертає нові `about/mission`;
  - `node scripts/smoke-check.mjs` — passed.

## Що покращило/виправило/додало

- Виправлено розсинхрон між адмінським save і публічним About: дані тепер коректно доходять до фронтенду.
- Усунуто джерело застарілих значень у `settings_i18n` для публічного рендера.
- Підвищено передбачуваність UI-оновлення About секції: DOM не зберігає старий шаблонний текст після зміни налаштувань.
