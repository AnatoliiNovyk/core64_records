# 490 Batch - Release year string normalization

## Як було

- При створенні або редагуванні релізу фронтенд відправляв поле `year` як число.
- Бекенд валідує `year` як рядок, тому API повертало 400 з помилкою валідації.
- У UI це виглядало як загальний alert: "Не вдалося зберегти запис. Перевірте дані і спробуйте ще раз."

## Що зроблено

- У [admin.js](admin.js#L3996) для `release.year` додано явне приведення результату `clampBoundedInteger(...)` до рядка через `String(...)`.
- Перевірено API сценарій: create (201), update (200), cleanup delete (204) для релізу з `year` у рядковому форматі.

## Що покращило

- Створення та редагування релізів більше не падає на серверній валідації типу `year`.
- Зникла хибна помилка збереження для валідних даних у формі релізу.
- Зменшено ризик повторної регресії через явну нормалізацію типу перед відправкою.
