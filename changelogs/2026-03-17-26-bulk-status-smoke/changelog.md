# Changelog 2026-03-17 #26 - Bulk status smoke

Після додавання масових дій потрібно було підтвердити, що множинні статус-оновлення стабільно проходять через API.

Створено 2 тестові звернення через `POST /api/contact-requests`.
Виконано авторизацію адміна.
Для обох ID виконано `PATCH /api/contact-requests/:id` зі статусом `done`.
Перевірено список `GET /api/contact-requests`.
Результат: `ids=3,4; statuses=done,done`.

Підтвердило коректність механіки, на якій базуються масові дії в UI.
Зменшило ризик помилок при пакетній обробці звернень.
Підтвердило узгодженість status-flow у backend та адмін-інтерфейсі.
