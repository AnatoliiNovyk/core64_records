# Changelog 2026-03-17 #32 - Логіка секції аудиту

## Як було
- У адмінці був лише UI-каркас аудиту без завантаження даних.

## Що було зроблено
- У `admin.js` додано:
  - `loadAuditLogs()` (читання з API за обраним лімітом),
  - `renderAuditLogs()` (пошук і рендер списку),
  - інтеграцію секції `audit` у `showSection()`.
- Додано `cache.auditLogs` для кешу журналу.

## Що це покращило, виправило, додало
- Додало реальний перегляд аудиторських подій у адмінці.
- Покращило контроль над операціями (хто/що/коли).
- Зменшило потребу у ручних API-запитах для перевірки аудиту.
