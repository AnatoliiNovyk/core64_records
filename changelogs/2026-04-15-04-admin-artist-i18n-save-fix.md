# Admin Artist I18n Save Fix

## Як було

- У модалці редагування артиста зміни тексту (наприклад `bio`) могли виглядати збереженими лише до перезавантаження секції.
- Після повторного відкриття запису повертався попередній переклад із `artists_i18n`, тому створювалось враження, що Save не спрацював.
- Причина: у CRUD-пayload при збереженні не передавався блок `i18n` для активної мови.

## Що зроблено

- У `admin.js` додано мапу перекладних полів для сутностей (`release`, `artist`, `event`, `sponsor`).
- Додано нормалізацію активної мови (`uk`/`en`) та helper, який формує/оновлює `item.i18n[activeLanguage]` перед create/update.
- У submit-потоці модалки збереження тепер відправляє payload із актуальними перекладами поточної мови разом із базовими полями.

## Що покращило/виправило/додало

- Текстові зміни в біографії артиста тепер коректно зберігаються в мовній таблиці і не відкатуються після перезавантаження.
- Збереження для локалізованих полів стало консистентним із читанням даних через API (`lang` + i18n fallback).
- Усунуто основну причину повідомленої проблеми “після Save повертається старий текст”.
