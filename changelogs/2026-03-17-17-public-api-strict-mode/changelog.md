# Changelog 2026-03-17 #17 - Public strict API mode

Публічна сторінка завжди тихо переходила на локальний fallback, навіть коли API недоступний.
Користувач не бачив явного статусу проблеми з backend.

У `index.html` додано контейнер `#public-api-status` для повідомлень про недоступний API.
У `index.html` додано `CORE64_CONFIG` з параметром `requireApiOnPublic`.
У `app.js` додано логіку:

- strict API для production-host за замовчуванням,
- показ явного повідомлення при недоступності API,
- fallback збережено лише для локального/дев-режиму.

Додало передбачувану поведінку для production: без тихого підмішування локальних даних.
Покращило UX/операційність: користувач бачить чітке повідомлення про інцидент API.
Зберегло зручність розробки локально через fallback.
