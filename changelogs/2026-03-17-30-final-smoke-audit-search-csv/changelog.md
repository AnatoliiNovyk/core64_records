# Changelog 2026-03-17 #30 - Фінальний smoke (audit + status flow)

Після додавання CSV/пошуку/аудиту потрібно було підтвердити, що весь контур працює в оновленому сервері.

Звільнено порт 3000 від старого процесу та перезапущено backend.
Створено тестове звернення через публічний endpoint.
Оновлено його статус через `PATCH /api/contact-requests/:id`.
Перевірено наявність аудиту через `GET /api/audit-logs`.
Результат: `createdId=6; auditFound=True; actor=admin; status=done`.

Підтвердило працездатність нового API читання аудиту.
Підтвердило наскрізний контур: submit -> status update -> audit event.
Закрило ризик регресії після останньої хвилі змін.
