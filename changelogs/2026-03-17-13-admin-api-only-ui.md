# Changelog 2026-03-17 #13 - Admin API-only mode + visible API error state

## Як було
- Адмінка могла тихо працювати у fallback-режимі, без явного індикатора недоступності API.
- Не було візуального банера зі статусом API.

## Що було зроблено
- У `admin.html` додано `window.CORE64_CONFIG` з `dataMode: "api"`.
- У `admin.html` додано контейнер `#api-status` для повідомлень про недоступний API.
- У `admin.js` додано:
  - `showApiStatus()` / `hideApiStatus()`;
  - стартову перевірку `adapter.isApiAvailable()`;
  - обробку помилок завантаження секцій із явним повідомленням користувачу.

## Що це покращило, виправило, додало
- Покращило керованість: адмінка тепер працює в strict API-only режимі.
- Виправило UX-проблему: при проблемі API користувач бачить чітку причину, а не "тиху" неконсистентність.
- Додало прозорий operational-flow для support/debug.
