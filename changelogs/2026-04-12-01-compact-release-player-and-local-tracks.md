# Compact Release Player And Local Tracks

## Як було

- Клік по релізу у публічній секції відкривав зовнішнє посилання, а не вбудований плеєр.
- Для релізів не існувало окремої моделі треків у БД та API-контракту для MP3/WAV завантажень.
- В адмінці не було інтерфейсу для керування кількома треками одного релізу.

## Що зроблено

- Додано міграцію `016_release_tracks.sql` з таблицею `release_tracks` (зв'язок з `releases`, сортування, тривалість, каскадне видалення).
- Розширено backend:
- Нові схеми валідації треків у `validate.js` (лише data URL для MP3/WAV).
- Нові методи репозиторію для читання і атомарної заміни треків релізу.
- Новий роут `releaseTracks.js` з ендпоінтами `GET /api/release-tracks/:releaseId` і `PUT /api/release-tracks/:releaseId`.
- Підключено роут у `server.js`.
- Розширено `data-adapter.js`:
- Додано локальний fallback для `releaseTracks`.
- Додано методи `getReleaseTracks(releaseId)` і `saveReleaseTracks(releaseId, tracks)`.
- Розширено адмін-панель:
- У модалку релізу додано блок керування треками (додати, видалити, пересунути).
- Upload тільки локального файлу MP3/WAV.
- Автоматичне зчитування тривалості.
- Валідація перед збереженням.
- Збереження треків разом із create/update релізу.
- Розширено публічну сторінку:
- Додано компактний вбудований плеєр у секцію релізів.
- Клік по картці або кнопці play відкриває плеєр.
- Треки підтягуються через API/fallback.
- Перший трек запускається автоматично.
- Підтримано ручний вибір треку зі списку та авто-перехід до наступного після завершення.

## Що покращило, виправило, додало

- Реалізовано сценарій прослуховування релізів без виходу з сайту через компактний плеєр.
- Додано повний цикл керування локально завантаженими треками MP3/WAV для кожного релізу.
- Вирівняно frontend/backend контракт для мульти-трекових релізів з API-first поведінкою і local fallback.
