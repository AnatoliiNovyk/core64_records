# 502 Batch - Спонсори, Партнери та Друзі (карусель)

## Як було

- На публічній сторінці не існувало секції для спонсорів і партнерів.
- В адмін-панелі не було CRUD для керування партнерськими картками.
- API/БД модель не містила сутність sponsors.

## Що зроблено

- Додано нову БД-міграцію `007_sponsors_table.sql` з таблицею `sponsors`.
- Розширено backend CRUD для `sponsors` у collections routes, repository і public payload.
- Додано валідацію sponsor-поля `shortDescription` з бізнес-правилом 3-5 слів.
- Оновлено seed і локальні fallback дані в data adapter для sponsors.
- На публічній сторінці додано секцію `Спонсори, Партнери та Друзі`.
- Реалізовано карусель логотипів з:
  - автопрокруткою;
  - паузою на hover/focus/hidden tab;
  - кнопками Prev/Next;
  - fallback-станом для порожнього списку.
- В адмін-панелі додано секцію Sponsors:
  - навігацію;
  - список карток;
  - форму create/edit/delete з полями name, shortDescription, logo, link, sortOrder.

## Що це покращило

- Партнерський блок став керованим через адмінку та БД, без ручного редагування HTML.
- Публічна сторінка отримала новий соціальний proof-блок у вигляді каруселі.
- Архітектура лишилась консистентною з існуючими collections (releases/artists/events).

## Додаткові правки після запуску

- Оновлено візуал карток sponsors у каруселі: картки зроблено квадратними з більш чіткою геометрією.
- Вирівняно пропорції контенту в картці sponsors до 25:75 (текст:зображення).
- В адмінці виправлено завантаження логотипа sponsor з комп'ютера:
  - для поля `logo` додано кнопку вибору файлу;
  - `handleFileUpload` тепер підтримує `input[name="logo"]`;
  - прев'ю логотипа оновлюється одразу після вибору файлу.
