# Changelog 2026-03-17 #28 - Backend аудит зміни статусів

Зміни статусів звернень не залишали серверного аудиту.
Міграції застосовували лише один SQL-файл (`001_init.sql`).

Додано міграцію `002_audit_logs.sql` з таблицею `audit_logs`.
Оновлено `migrate.js` на послідовне застосування всіх `.sql`-міграцій.
Додано репозиторні методи `writeAuditLog()` та `listAuditLogs()`.
У `PATCH /api/contact-requests/:id` додано запис аудиту `status_updated`.

Додало простежуваність критичних адмін-дій (хто/коли/що змінив).
Виправило технічний борг у міграційному механізмі (підтримка кількох версій).
Підготувало базу для майбутнього екрану журналу дій.
