# 415 Batch - FileReader loadTarget result guard

як було:

- У `reader.onload` значення base64 бралося напряму через `e.target.result` без перевірки структури payload події.

що зроблено:

- Додано захисну валідацію `loadTarget` у `reader.onload`.
- Додано ранній вихід, якщо `loadTarget` відсутній або `loadTarget.result` не є рядком.
- Присвоєння `base64` виконується тільки після успішної валідації.

що покращило/виправило/додало:

- Усунено ризик runtime-помилки в рідкісних edge-case сценаріях з невалідним об'єктом події FileReader.
- Підвищено стабільність upload-потоку без зміни бізнес-логіки та UI-поведінки.
