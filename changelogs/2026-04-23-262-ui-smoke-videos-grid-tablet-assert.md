# 2026-04-23-262 UI Smoke Videos Grid Tablet Assert

## Як було
- У UI smoke вже була перевірка 4-колонкового desktop-режиму секції Video.
- Перевірки tablet-режиму (2 колонки) не було, тому брейкпоінт `md` міг регреснути непомітно.

## Що зроблено
- У `scripts/ui-smoke.mjs` в `verifyPublicUi` додано окрему tablet-перевірку:
  - встановлення viewport `1024x900`,
  - перевірка наявності `#videos-grid`,
  - перевірка класу `md:grid-cols-2`,
  - перевірка фактичної кількості колонок через `gridTemplateColumns` = 2.
- Після tablet-асерту viewport повертається на desktop (`1920x1080`) для наступних перевірок.

## Що покращило/виправило/додало
- Закрито матрицю ключових брейкпоінтів для секції Video в UI smoke: tablet=2, desktop=4.
- Зменшено ризик непомітної регресії адаптивної сітки після майбутніх UI-змін.
