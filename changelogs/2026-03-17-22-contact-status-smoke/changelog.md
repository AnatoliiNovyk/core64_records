# Changelog 2026-03-17 #22 - Smoke тест зміни статусу звернення

## Як було
- Після впровадження PATCH-ендпоінта та UI-контролів статусу потрібно було підтвердити e2e-роботу.

## Що було зроблено
- Запущено backend (`npm run dev`).
- Створено тестове звернення через `POST /api/contact-requests`.
- Виконано авторизацію адміна.
- Виконано `PATCH /api/contact-requests/:id` зі статусом `in_progress`.
- Перевірено `GET /api/contact-requests`, що статус оновився.
- Результат: `id=2; patchedStatus=in_progress; listStatus=in_progress`.

## Що це покращило, виправило, додало
- Підтвердило наскрізну працездатність нового статусного контуру.
- Підтвердило, що адмін-фільтри/керування статусом мають валідні дані з API.
- Закрило ризик розсинхрону між PATCH-оновленням і списком звернень.
