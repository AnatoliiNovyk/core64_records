# 501 Batch - Повна дата релізу замість року

## Як було

- У релізів основним полем відображення в UI був лише рік.
- У формі релізу в адмінці використовувалось текстове поле року.
- Бекенд модель не вимагала повну дату релізу.

## Що зроблено

- У формі релізу в адмінці поле року замінено на обов'язкове поле дати `releaseDate` (`type=date`).
- У списку релізів адмінки та на публічній сторінці показується повна дата у форматі `DD.MM.YYYY`.
- У backend валідацію релізу додано обов'язковий `releaseDate` у форматі `YYYY-MM-DD`.
- У репозиторії БД додано підтримку `release_date` з мапінгом у `releaseDate`.
- Додано SQL міграцію `006_releases_release_date.sql` з backfill існуючих рядків на основі `year`.
- Оновлено seed і локальні дефолтні дані з повною датою релізу.

## Що це покращило

- Релізи тепер мають точну дату замість лише року.
- UI, API і БД використовують узгоджену модель повної дати релізу.
- Існуючі дані автоматично мігруються і залишаються сумісними через синхронізацію `year`.
