# 2026-04-16-29 Admin Requests Attachment Visibility And Download

## Як було

- У розділі `Requests` адмінка показувала тему, текст і статус звернення, але не відображала вкладений demo-файл.
- Навіть якщо `attachmentDataUrl` зберігався у БД, в UI не було жодної кнопки для відкриття або завантаження вкладення.
- Експорт звернень у CSV не містив ознак наявності вкладення.

## Що зроблено

- В `admin.js` додано нормалізацію вкладень для contact requests (`name`, `type`, `dataUrl`, `hasData`, `isAudio`).
- Додано безпечні helper-функції для роботи з data URL:
- перетворення data URL у `Blob`
- визначення розширення файла за MIME
- формування fallback-імені файла
- Додано дії в адмінці для кожного звернення з вкладенням:
- `Open` (для аудіо)
- `Download` (для будь-якого вкладення)
- Додано рендер attachment-блоку в картці звернення:
- назва файла
- тип файла
- кнопки дій або попередження про відсутність payload
- Оновлено локалізацію (uk/en) для нових підписів і помилок attachment-дій.
- Розширено CSV-експорт звернень колонками:
- `attachment_name`
- `attachment_type`
- `has_attachment`

## Що покращило/виправило/додало

- Адмін тепер бачить, чи у зверненні є вкладення, і може відкрити/завантажити demo-файл прямо з `Requests`.
- Усунуто UX-розрив, коли в БД вкладення було, але в інтерфейсі воно виглядало як "відсутнє".
- CSV-експорт став придатним для швидкої технічної перевірки наявності вкладень.
