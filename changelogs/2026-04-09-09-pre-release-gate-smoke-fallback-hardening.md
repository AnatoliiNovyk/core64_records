# 2026-04-09-09 Pre Release Gate Smoke Fallback Hardening

## Як було

- Локальний gate міг падати на smoke через нестабільний `localhost` резолв (`fetch failed`).
- Доданий раніше retry на `127.0.0.1` спрацьовував на будь-яку помилку smoke, навіть якщо причина не мережна.
- Скрипт залежав від поточної директорії запуску.
- Для `CORE64_ADMIN_PASSWORD` використовувався параметр за замовчуванням `core64admin`, що могло конфліктувати з фактичним `ADMIN_PASSWORD` у `backend/.env`.

## Що зроблено

- У [scripts/pre-release-gate-local.ps1](scripts/pre-release-gate-local.ps1) додано прив'язку до кореня репозиторію через `$PSScriptRoot`, щоб відносні шляхи працювали незалежно від `cwd`.
- Додано резолвер пароля адміна:
  - пріоритет 1: `CORE64_ADMIN_PASSWORD` з середовища;
  - пріоритет 2: явно переданий `-Core64AdminPassword` (якщо не дефолтний `core64admin`);
  - пріоритет 3: `ADMIN_PASSWORD` з `backend/.env`;
  - fallback: `core64admin`.
- Крок smoke винесено в окрему функцію з контрольованим виводом та результатом (`ExitCode`, `Output`).
- Retry з `localhost` на `127.0.0.1` тепер виконується тільки коли smoke впав саме з `fetch failed`.

## Що покращило/виправило/додало

- Gate став стійкішим у локальному середовищі без зайвих повторних запусків smoke при немережних помилках.
- Усунуто залежність від каталогу запуску скрипта.
- Знижено ризик хибних падінь через розсинхрон `Core64AdminPassword` vs `backend/.env`.
- Збережено fail-fast поведінку для реальних smoke/regression проблем.
