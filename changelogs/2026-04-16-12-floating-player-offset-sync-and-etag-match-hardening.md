# Floating Player Offset Sync And ETag Match Hardening

## Як було

- У floating-режимі нижній службовий відступ сторінки напряму залежав від повної висоти плеєра, через що внизу могла з'являтися занадто велика порожня зона.
- Перерахунок офсету виконувався переважно через `resize`, тому окремі зміни висоти контенту плеєра могли синхронізуватися із затримкою.
- Перевірка `If-None-Match` для audio endpoint була занадто вузькою: очікувала точне співпадіння одного значення.

## Що зроблено

- У `app.js` розділено два офсети:
  - `--release-player-offset` залишено для позиціонування нижніх floating-кнопок вище плеєра.
  - `--release-player-body-padding` додано окремо для контрольованого `padding-bottom` сторінки з м'якшим масштабуванням.
- Додано `ResizeObserver` для кореневого елемента release player, щоб зміни висоти в floating-стані одразу оновлювали CSS-перемінні офсету.
- У `index.html` оновлено правило `body.has-floating-release-player`: тепер використовується `--release-player-body-padding` замість повної висоти плеєра.
- У `backend/src/routes/releaseTracks.js` додано більш надійне зіставлення `If-None-Match`:
  - підтримка списку значень через кому,
  - підтримка wildcard `*`,
  - weak/strong нормалізація ETag для порівняння.

## Що покращило/виправило/додало

- При активному floating player прибрано надмірну порожню область у нижній частині сторінки без втрати клікабельності допоміжних кнопок.
- Синхронізація відступів стала стабільнішою при динамічній зміні висоти плеєра.
- Кеш-валидація audio endpoint стала більш сумісною з реальними HTTP-клієнтами, які надсилають складні значення `If-None-Match`.
