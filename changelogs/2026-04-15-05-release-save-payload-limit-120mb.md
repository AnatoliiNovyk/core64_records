# Release Save Payload Limit 120MB

## Як було

- Backend приймав JSON payload до `60mb`.
- При збереженні релізу з кількома локальними треками (base64 data URL) сумарний обсяг запиту міг перевищувати цей ліміт.
- У результаті з'являлась помилка `Payload is too large to save` (HTTP 413), і збереження релізу не виконувалось.

## Що зроблено

- У `backend/src/server.js` підвищено ліміт `express.json` з `60mb` до `120mb`.
- Логіка обробки 413 (`PAYLOAD_TOO_LARGE`) збережена без змін як захист від надмірних запитів.

## Що покращило/виправило/додало

- Збереження релізів з кількома MP3/WAV треками стало проходити у випадках, де раніше запит перевищував 60MB.
- Зменшено кількість хибних блокувань на етапі save при валідному контенті.
- Контроль верхньої межі payload залишається на backend, але з практичнішим порогом для реального сценарію використання.
