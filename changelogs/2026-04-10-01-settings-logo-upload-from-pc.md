# Changelog 2026-04-10 #01 - Settings logos upload from PC

## Як було
- Логотипи хедера та футера в налаштуваннях адмінки можна було змінювати лише через URL або відносний шлях.
- Вставка зображення з локального комп'ютера безпосередньо в налаштування була відсутня.
- На публічній сторінці нормалізація логотипів відхиляла data URL, тому upload-сценарій не підтримувався.

## Що було зроблено
- У блоці налаштувань брендингу додано кнопки завантаження файлу для header/footer логотипів, приховані file input та прев'ю зображень.
- Реалізовано обробник завантаження в адмінці з перевіркою формату та розміру файлу, читанням через FileReader і записом у поле налаштування як data URL.
- Додано оновлення прев'ю під час ручного редагування поля та після завантаження.
- Оновлено i18n-підписи/підказки для нового сценарію завантаження.
- На публічній частині дозволено безпечні data:image/*;base64,... значення для логотипів, із блокуванням небезпечних схем.
- У backend валідації збільшено допустиму довжину для headerLogoUrl/footerLogoUrl, щоб приймати base64 payload після upload.
- Прогнано перевірки: API smoke, UI smoke, settings/public contract check, i18n consistency.

## Що це покращило, виправило, додало
- Додано запитану можливість змінювати логотипи без хостингу файлів за URL: адміністратор може завантажити файл прямо з комп'ютера.
- Зменшено операційний бар'єр для оновлення брендингу та прискорено рутинні зміни в адмінці.
- Збережено безпеку рендеру логотипів через обмеження на допустимі data URL формати.
- Підтверджено, що новий сценарій не ламає існуючі API/UI контракти.
