# 416 Batch - Modal submit form tag guard

- як було:
  - У submit-обробнику модалки виконувалася лише перевірка на існування/підключеність `event.target`, після чого одразу створювався `FormData`.

- що зроблено:
  - Додано додаткову валідацію типу submit-target: обробник продовжується тільки якщо `tagName` існує і дорівнює `FORM`.
  - `new FormData(formEl)` тепер викликається лише для валідного HTML form-елемента.

- що покращило/виправило/додало:
  - Зменшено ризик runtime-помилки в edge-case сценаріях з неочікуваним retargeting події.
  - Підвищено стабільність submit-потоку модалки без зміни поточної бізнес-логіки.
