# Changelog - 2026-03-17 - Audit rolling latency average

Додано відображення середньої тривалості refresh по останніх 5 запитах аудиту.
Збережено існуючу метрику останньої тривалості, додано другу метрику для стабільності.

Файл: `admin.html`

- У статусному блоці аудиту додано:
- `#audit-avg-latency` з початковим текстом `Середня (останні 5): -`.

Файл: `admin.js`

- Додано стани:
- `auditLatencyHistory = []`
- `AUDIT_LATENCY_HISTORY_SIZE = 5`
- У `loadAuditLogs()`:
- обчислюється `latencyMs` для поточного завантаження;
- метрика додається в history (обрізка до останніх 5 значень);
- обчислюється rolling average і оновлюється `#audit-avg-latency`.

Статичні перевірки:

- `admin.js`: без помилок
- `admin.html`: без помилок

Smoke API:

- `okPage=1; okTotal=5; okItems=1`

В адмінці видно не лише пікове/останнє значення затримки, а й середню швидкість відповіді за коротке вікно, що краще відображає реальну стабільність refresh.
