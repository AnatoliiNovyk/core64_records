# 429 Batch - Edit item non-finite number id guard

як було:

- `editItem` валідовував `type`, `null/undefined` і порожній строковий `id`, але не відсікав невалідні числові значення (`NaN`, `Infinity`).

що зроблено:

- Додано ранній guard: якщо `id` має тип `number` і `Number.isFinite(id)` повертає `false`, функція завершується.

що покращило/виправило/додало:

- Усунуто edge-case виклики редагування з некоректним числовим ідентифікатором.
- Підвищено стабільність edit-flow без змін штатної поведінки для валідних id.
