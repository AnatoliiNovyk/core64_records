# 445 Batch - OpenModal cache entry null guard

- як було:
  - У `openModal` пошук запису в `collectionItems.find(...)` очікував коректні об'єкти елементів.
  - За атипового стану масиву (наприклад, `null/undefined` елементи) доступ до `entry.id` міг бути крихким.

- що зроблено:
  - Додано захисну перевірку елемента всередині `find`:
    - якщо елемент відсутній або не є об'єктом, він пропускається.
  - Порівняння id виконується лише для валідних object-елементів.

- що покращило/виправило/додало:
  - Підвищено стійкість `openModal` до пошкодженого вмісту кеш-масиву.
  - Усунуто edge-case ризик помилки при lookup у нестандартному стані.
