# 2026-04-21-250 Public Outbound Link Sanitization

## Як було

- У публічному рендері частина outbound-посилань підставлялась напряму з даних без нормалізації URL: artists social links, event ticket links, sponsor links.
- У разі невалідної або небезпечної схеми посилання могло некоректно рендеритись і відкриватись у браузері.

## Що зроблено

- Для artists/events/sponsors у app.js підключено централізовану нормалізацію через `normalizeReleaseOutboundUrl(...)` перед формуванням href.
- Для валідних посилань збережено відкриття у новій вкладці (`target="_blank" rel="noopener noreferrer"`).
- Для невалідних/порожніх посилань додано безпечний fallback без зовнішнього переходу.

## Що покращило/виправило/додало

- Знижено ризик небезпечних схем у публічних outbound href.
- Підвищено стабільність UI при частково битих або неочікуваних URL у даних.
- Уніфіковано поведінку публічних посилань через єдиний URL-normalization шлях.
