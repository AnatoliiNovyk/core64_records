# 422 Batch - Delete item type allowlist guard

як було:

- `deleteItem` не перевіряв `type` на належність до підтримуваних сутностей перед викликом adapter delete.

що зроблено:

- Додано ранній allowlist-guard: виконання продовжується тільки для `release`, `artist`, `event`.

що покращило/виправило/додало:

- Усунуто edge-case ризик виклику видалення для невалідного типу.
- Підвищено передбачуваність і безпечність delete-flow без змін штатної поведінки.
