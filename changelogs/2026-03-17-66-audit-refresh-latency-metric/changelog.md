# Changelog - 2026-03-17 - Audit refresh latency metric

## Що зроблено
- Додано метрику останньої тривалості завантаження аудиту (у ms) у статус-блок секції аудиту.
- Метрика оновлюється після кожного успішного `loadAuditLogs()` (ручний refresh, автооновлення, фільтрація).

## Технічні зміни
- Файл: `admin.html`
  - У статусному блоці додано елемент:
    - `#audit-last-latency` з початковим текстом `Остання тривалість: -`.

- Файл: `admin.js`
  - У `loadAuditLogs()` додано вимір часу запиту:
    - старт: `performance.now()` (fallback: `Date.now()`)
    - завершення: `performance.now()` (fallback: `Date.now()`)
  - Після успішного завантаження оновлюється `#audit-last-latency`:
    - `Остання тривалість: <N> ms`.

## Валідація
- Статичні перевірки:
  - `admin.js`: без помилок
  - `admin.html`: без помилок
- Smoke API:
  - `okPage=1; okTotal=5; okItems=1`

## Результат
- В адмінці тепер видно фактичну тривалість останнього refresh аудиту, що допомагає швидко оцінити продуктивність запитів у реальному часі.
