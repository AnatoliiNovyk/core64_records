# 417 Batch - File upload files list guard

- як було:
  - У `handleFileUpload` читання файлу виконувалося напряму через `input.files[0]` без окремої перевірки, що `files` існує та містить елементи.

- що зроблено:
  - Додано проміжну змінну `files`.
  - Додано ранній вихід, якщо `files` відсутній або порожній.
  - Ініціалізація `file` тепер виконується лише після валідації списку файлів.

- що покращило/виправило/додало:
  - Усунуто edge-case ризик помилки доступу до `files[0]` при невалідному/порожньому `input.files`.
  - Підвищено надійність upload-флоу без змін бізнес-логіки та UI-поведінки.
