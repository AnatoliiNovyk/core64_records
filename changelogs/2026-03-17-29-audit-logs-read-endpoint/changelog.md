# Changelog 2026-03-17 #29 - API читання audit logs

Аудит записувався в БД, але не мав API для контрольного читання/перевірки.

Додано маршрут `GET /api/audit-logs` (під auth) у `routes/auditLogs.js`.
Додано підключення цього маршруту в `server.js`.
Додано підтримку `limit` query-параметра (до 500).

Додало штатний спосіб перевіряти аудиторські події через API.
Прибрало залежність від локальних драйверів/прямих SQL-перевірок для smoke.
Підготувало основу для потенційної UI-сторінки журналу дій.
